## Lessons Learned

- Cloudinary presets make image management consistent
- Local images need to be organized before upload
- Public ID structure should mirror local folder structure
- Use `next-cloudinary` for Next.js Image component compatibility

## Best Practices Identified

- Organize images by project in Cloudinary folders
- Use consistent naming conventions (e.g., `project-name/hero`)
- Implement progressive loading with srcset
- Store Cloudinary public IDs in database, not full URLs
